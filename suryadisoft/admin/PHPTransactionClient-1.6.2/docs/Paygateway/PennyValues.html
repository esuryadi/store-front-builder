PHP Transaction Client v1.6.1 API Specification

Penny Value Tests

Before using the PHP Transaction Client on a production system, you should test your system's response to the various response codes that the PHP Transaction Client may return. This can be done without sending actual transaction information by using penny values. Testing in this mode allows you to verify that the correct response codes are being generated by your system without the transaction information actually being sent to your bank account.

To perform the penny value tests, follow the steps below:

  1. Ensure the prefix TEST is located on the front of your account token.
  2. Successfully perform each of the tests in the table below with the indicated penny amounts. Use the credit card number 5454545454545454 with an expiry date of 10/2002.
  3. Repeat the tests in step 2, for all the transaction types your payment system will be using (Sale, Auth, Capture, Void, Credit).

For each test do the following:

  • Validate that all the responses are handled properly for each transaction type.
  • Verify the correct order confirmation response codes.
  • Validate that all required fields are incorporated.
  • Ensure that the required fields for each transaction type are passed, and handled properly.
  • Ensure that all information on the order confirmation page is displayed properly and that all order confirmation page variables are shown.
